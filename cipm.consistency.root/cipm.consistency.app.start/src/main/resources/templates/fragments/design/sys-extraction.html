System model building process
Using this tool it is possible to build system models in a semi-automated (user-driven) process. This tool requires an existing and configured repository, which for example can be generated with EJBMox or manually by hand.
The semi-automated extraction of a system model requires user input to resolve conflicts. It is recommended to build a service call graph (SCG) before the extraction. This way, the number of conflicts and thus the degree of user interaction required can be reduced to a minimum.
The current service-call-graph (SCG) contains [[${scg_nodes}]] nodes and [[${scg_edges}]] edges.
